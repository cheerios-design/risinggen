📋 Authentication Overview

RisingGen uses JWT (JSON Web Token) authentication integrated with Church of Jesus Christ of Latter-day Saints membership verification. This ensures that only verified YSA members can access the platform while maintaining secure API access.

Key Features

  • LDS Membership Integration: Verification through membership numbers
  • JWT Tokens: Secure, stateless authentication
  • Role-Based Access: Different permission levels for users
  • Multi-Language Support: Authentication in user's preferred language
  • Session Management: Configurable token expiration

🎟️ JWT Token Authentication

Login Process

Users authenticate using their LDS membership number and password:

POST
/api/auth/login

Authenticate user with membership credentials

Request Format

POST /api/auth/login Content-Type: application/json { "membershipNumber": "123456789", "password": "securePassword123", "language": "en" }

Success Response

200 OK
{ "success": true, "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...", "user": { "id": "user_123", "name": "John Doe", "email": "john.doe@example.com", "preferredLanguage": "en", "role": "participant", "ward": "London 1st Ward", "stake": "London England Stake" }, "expiresIn": 3600, "refreshToken": "refresh_abc123..." }

Using JWT Tokens

Include the JWT token in the Authorization header for all API requests:

Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...

Token Refresh

POST
/api/auth/refresh

Refresh expired token using refresh token

POST /api/auth/refresh Content-Type: application/json { "refreshToken": "refresh_abc123..." }

👥 User Roles and Permissions

The system supports four main user roles with different permission levels:

Role Permissions Description
participant Register for events, connect with others, view content Regular YSA member
organizer Create events, manage registrations, moderate groups Event organizer with management permissions
leadership Oversee regional activities, access reports, approve events Area/regional leadership
admin Full system access, user management, platform configuration System administrator

Role Assignment

User roles are automatically assigned based on:

  • Church Callings: Leadership roles reflected in permissions
  • Event History: Frequent organizers may receive organizer role
  • Manual Assignment: Administrators can manually assign roles
  • Temporary Permissions: Event-specific permissions can be granted

💻 Code Examples

JavaScript/Node.js

// Login function async function loginUser(membershipNumber, password, language = 'en') { try { const response = await fetch('/api/auth/login', { method: 'POST', headers: { 'Content-Type': 'application/json', }, body: JSON.stringify({ membershipNumber, password, language }) }); const data = await response.json(); if (data.success) { // Store token securely localStorage.setItem('authToken', data.token); localStorage.setItem('refreshToken', data.refreshToken); return data.user; } else { throw new Error(data.error.message); } } catch (error) { console.error('Login failed:', error); throw error; } } // Authenticated API request async function makeAuthenticatedRequest(url, options = {}) { const token = localStorage.getItem('authToken'); const response = await fetch(url, { ...options, headers: { ...options.headers, 'Authorization': `Bearer ${token}`, 'Content-Type': 'application/json' } }); // Handle token expiration if (response.status === 401) { await refreshToken(); // Retry request with new token return makeAuthenticatedRequest(url, options); } return response; }

import requests import json from datetime import datetime, timedelta class RisingGenAuth: def __init__(self, base_url="https://api.risinggen.eu"): self.base_url = base_url self.token = None self.refresh_token = None self.expires_at = None def login(self, membership_number, password, language="en"): """Authenticate user with membership credentials""" url = f"{self.base_url}/api/auth/login" payload = { "membershipNumber": membership_number, "password": password, "language": language } response = requests.post(url, json=payload) data = response.json() if data.get("success"): self.token = data["token"] self.refresh_token = data["refreshToken"] self.expires_at = datetime.now() + timedelta(seconds=data["expiresIn"]) return data["user"] else: raise Exception(data.get("error", {}).get("message", "Login failed")) def get_headers(self): """Get headers with authentication token""" if not self.token or datetime.now() >= self.expires_at: self.refresh_access_token() return { "Authorization": f"Bearer {self.token}", "Content-Type": "application/json" } def refresh_access_token(self): """Refresh the access token""" if not self.refresh_token: raise Exception("No refresh token available") url = f"{self.base_url}/api/auth/refresh" payload = {"refreshToken": self.refresh_token} response = requests.post(url, json=payload) data = response.json() if data.get("success"): self.token = data["token"] self.expires_at = datetime.now() + timedelta(seconds=data["expiresIn"]) else: raise Exception("Token refresh failed")

🔒 Security Best Practices

⚠️ Security Notice

Never store JWT tokens in localStorage on production applications. Use secure, httpOnly cookies or session storage instead.

Token Security

  • Secure Storage: Store tokens in httpOnly cookies or secure session storage
  • HTTPS Only: Always use HTTPS for token transmission
  • Token Expiration: Implement appropriate token expiration times
  • Refresh Tokens: Use refresh tokens to minimize long-lived access tokens
  • Token Validation: Always validate tokens on the server side

Password Security

  • Strong Passwords: Enforce strong password requirements
  • Rate Limiting: Implement login attempt rate limiting
  • Account Lockout: Lock accounts after multiple failed attempts
  • Two-Factor Authentication: Optional 2FA for enhanced security

API Security

  • Rate Limiting: Implement per-user and global rate limits
  • Input Validation: Validate all input parameters
  • CORS Configuration: Properly configure CORS policies
  • Audit Logging: Log all authentication and authorization events

⚠️ Error Handling

Authentication Errors

401 Unauthorized
{ "success": false, "error": { "code": "INVALID_CREDENTIALS", "message": "Invalid membership number or password", "details": { "attemptCount": 2, "maxAttempts": 5, "lockoutTime": null } } }
401 Unauthorized
{ "success": false, "error": { "code": "TOKEN_EXPIRED", "message": "JWT token has expired", "details": { "expiredAt": "2023-07-15T15:30:45Z", "canRefresh": true } } }

Common Error Codes

Code Description Action
INVALID_CREDENTIALS Wrong membership number or password Prompt user to retry
TOKEN_EXPIRED JWT token has expired Use refresh token
TOKEN_INVALID Malformed or invalid token Redirect to login
ACCOUNT_LOCKED Too many failed login attempts Show lockout message
MEMBERSHIP_NOT_FOUND Membership number not found Verify membership details
